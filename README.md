# Папа рядом

Android-приложение для отца и семьи во время подготовки к родам, в самих родах, в роддоме и в первые дни дома после выписки.

Приложение остается `offline-first`: все ключевые данные хранятся локально, а интерфейс перестраивается вокруг текущего этапа и реального сценария, а не вокруг набора разрозненных инструментов.

## Что сейчас реализовано

- 4-stage модель приложения: `PREPARING`, `CONTRACTIONS`, `AT_HOSPITAL`, `AT_HOME`
- ручное и сценарное управление этапами
- строгие domain guards для нелогичных переходов
- контекстная `Главная` с live-блоком схваток
- stage-aware экран `События`
- `Журнал` с компактной историей и раскрытием деталей
- системные и пользовательские чек-листы
- user-managed контакты с адресом роддома и построением маршрута
- `SOS` с быстрыми звонками
- Room + DataStore для локального хранения

## Этапы приложения

Поддерживаются четыре этапа:

- `PREPARING` / `Подготовка`
- `CONTRACTIONS` / `Схватки`
- `AT_HOSPITAL` / `В роддоме`
- `AT_HOME` / `Дома`

### Важные правила

- ПДР не переводит приложение между этапами автоматически.
- Ручное переключение этапа возможно, но после фиксации рождения запрещен откат в `PREPARING` и `CONTRACTIONS`.
- Действие `Начались роды` блокируется, если рождение уже сохранено.
- Действие `Приехали домой` разрешено только после фиксации рождения и только из hospital-сценария.

## Логика ПДР

ПДР используется только как контекстный ориентир:

- если ПДР не заполнена, на `Главной` каждый запуск показывается reminder;
- до 37 недель `Главная` держит фокус на подготовке;
- с 37 недель на `Главной` появляются hospital/labor-ready блоки;
- ПДР влияет на приоритет карточек, но не на stage transition.

## Главная

`Главная` отвечает на вопросы:

- что происходит сейчас
- что важно сделать дальше
- какие быстрые действия доступны сразу

### Поведение по этапам

`PREPARING`

- подготовительные карточки
- ПДР и readiness-context
- чек-листы этапа
- мягкий вход в счетчик схваток ближе к ПДР

`CONTRACTIONS`

- live-блок схваток наверху
- интервал, текущий статус и рекомендация
- таймер вод
- спокойный блок быстрых действий: контакты, маршрут, звонок врачу / роддому
- релевантные чек-листы ниже

`AT_HOSPITAL`

- счетчик схваток убирается
- на первый план выходят данные родов, журнал и hospital-задачи
- если таймер вод еще активен, доступ к нему сохраняется

`AT_HOME`

- родовые боевые инструменты скрыты
- на первый план выходят домашние события, трекеры, помощь маме и чек-листы дома

## События

`События` — это рабочий экран действий по текущему этапу.

Примеры:

- `PREPARING`: заметки, готовность сумки, тестовый запуск счетчика
- `CONTRACTIONS`: схватки, воды, выезд, приезд, начало родов
- `AT_HOSPITAL`: рождение, поддержка, hospital notes, переход домой
- `AT_HOME`: кормление, сон, подгузник, температура, вес, заметки

Логика схваток переиспользуется из одного источника правды и одинакова на `Главной` и на профильном экране схваток.

## Журнал

`Журнал` показывает историю событий.

Что хранится:

- старт и завершение схваток
- события по водам
- начало родов
- выезд и приезд
- рождение ребенка
- возвращение домой
- ручные заметки по этапам

В общей истории элементы компактные по умолчанию и раскрываются по нажатию. Для схваток в раскрытии видны:

- начало
- конец
- длительность
- интервал

На отдельном экране схваток детальная история остается развернутой всегда.

## Чек-листы

Поддерживаются:

- системные чек-листы
- пользовательские чек-листы

Что можно делать:

- создавать свой список в любом этапе
- переименовывать пользовательский список
- добавлять пункт
- удалять пункт
- отмечать пункт выполненным / невыполненным
- удалять пользовательский список

По умолчанию экран фильтруется по текущему этапу, но фильтр можно переключать вручную.

## Экстренные контакты и SOS

Контакты теперь полностью user-managed.

По умолчанию создается только:

- `112`

Пользователь может добавлять:

- жену
- врача
- роддом
- такси
- родственника
- произвольный контакт

Для контакта роддома поддерживаются:

- звонок
- маршрут
- копирование адреса

`SOS` использует эти контакты как источник быстрых звонков:

- `112`
- врач, если добавлен
- жена, если добавлена
- остальные быстрые контакты через карточку контактов

## Хранение данных

Используется локальное хранение:

- `Room` для событий, истории, чек-листов, labor summary, трекеров и контактов
- `DataStore` для пользовательских настроек и app-level stage

В `Settings` поле адреса роддома оставлено только ради совместимости старых сохранений. Актуальный адрес для маршрута теперь хранится в контактах.
