# pap_app

`pap_app` — offline-first Android-приложение-помощник для отца вокруг родов и первых дней после них. Приложение перестраивает интерфейс вокруг текущего этапа семьи, а не вокруг набора разрозненных инструментов.

## Что важно сейчас
- 3-stage модель приложения: `PREPARING`, `LABOR`, `BABY_BORN`
- локальное хранение через `Room` и `DataStore`
- единый сценарий `Активные роды` для схваток, вод и рекомендации
- журнал с привязкой записи к этапу создания
- системные и пользовательские чек-листы с возможностью удаления
- нижняя навигация: `Главная`, `События`, `Чек-листы`, `Контакты`

## Модель этапов
Поддерживаются только три этапа:
- `PREPARING` / `Подготовка`
- `LABOR` / `Активные роды`
- `BABY_BORN` / `Ребенок родился`

Этап — это состояние приложения, а не физическое место.

Автоматические переходы:
- первая запущенная схватка переводит приложение в `LABOR`
- фиксация вод переводит приложение в `LABOR`
- фиксация рождения переводит приложение в `BABY_BORN`

Ручной переход остается доступным через экран этапа из бокового меню. Выбор этапа делается не прямым тапом по названию, а через отдельный preview screen с кнопкой подтверждения.

## Главная
### PREPARING
На `Главной` показываются:
- напоминание заполнить ПДР, если она отсутствует
- подготовительный блок и чек-листы этапа
- информационный блок
- последние события

### LABOR
На `Главной` приоритет такой:
1. live-блок `Активные роды`
2. статус вод
3. быстрый доступ к контактам
4. чек-лист этапа
5. последние события

### BABY_BORN
На `Главной` сверху всегда показывается карточка ребенка, даже если имя, вес и рост еще не заполнены.
Ниже идут:
- приоритетные чек-листы после рождения
- остальные чек-листы этапа
- последние события

## Экран «Активные роды»
Экран объединяет то, что раньше было разнесено между счетчиком схваток и отдельными действиями.

Содержимое:
- короткая рекомендация
- статус схватки
- длительность и интервал
- статус вод
- действия:
  - старт или стоп схватки
  - запись вод
  - `Ребенок родился`

Приоритет рекомендаций:
1. если рождение уже зафиксировано, родовые рекомендации больше не главные
2. если воды отошли и они не прозрачные, нужен срочный контакт
3. если воды прозрачные, нужен контакт с врачом или роддомом и наблюдение
4. если воды не отходили, используется текущий алгоритм интервалов схваток

## События и журнал
`События` — рабочий экран действий по текущему этапу.

Примеры по этапам:
- `PREPARING`: заметки подготовки, пробный выезд, готовность сумки, вход в `Активные роды`
- `LABOR`: старт или стоп схватки, воды, рождение, логистика, заметки этапа
- `BABY_BORN`: карточка ребенка, уходовые записи, заметки, быстрые действия после рождения

Журнал хранит:
- системные события
- пользовательские заметки

У каждой записи есть:
- `timestamp`
- `stageAtCreation`
- `entryType`
- `type`
- `title`
- `description`

Фильтры журнала:
- `Все`
- `Подготовка`
- `Активные роды`
- `Ребенок родился`
- `Мои заметки`

`stageAtCreation` записывается в момент создания записи и потом не меняется.

## Чек-листы
Поддерживаются:
- системные чек-листы
- пользовательские чек-листы

Пользователь может:
- создать чек-лист в любом этапе
- переименовать пользовательский чек-лист
- добавить пункт
- удалить пункт
- отметить пункт выполненным или невыполненным
- удалить любой чек-лист

Если системный чек-лист удален, он не создается заново автоматически.

## Контакты
Экран `Контакты` доступен из нижней навигации.

По умолчанию есть:
- `Скорая помощь`
- `Роддом`

Дополнительно можно добавлять пользовательские контакты.
Поддерживаются:
- звонок
- редактирование телефона
- хранение адреса роддома
- маршрут до роддома

## Удаления и пересчет состояния
Поддерживаются подтвержденные удаления:
- записи о водах
- отдельной схватки
- чек-листа
- пункта чек-листа

После удаления:
- схватка исключается из расчета интервалов и рекомендации
- удаленная запись о водах перестает влиять на рекомендацию
- удаленный системный чек-лист не восстанавливается автоматически
