# Папа рядом

Android-приложение для будущего отца и семьи во время подготовки к родам, в процессе родов и сразу после рождения ребенка.

Приложение остается `offline-first`: ключевые данные хранятся локально на устройстве, а интерфейс строится вокруг текущего этапа, а не вокруг набора разрозненных инструментов.

## Что изменилось в продукте

- Навигация перестроена вокруг понятных человеку разделов.
- Появилась централизованная модель этапов приложения: `PREPARING`, `LABOR`, `AFTER_BIRTH`.
- ПДР используется как контекстный ориентир, но не переключает этап автоматически.
- Раздел `Timeline` переосмыслен в пользовательский `Журнал`.
- Добавлены ключевые события `Начались роды` и `Ребенок родился`.
- Главный экран стал контекстной панелью состояния.
- Чек-листы переработаны и разбиты по этапам.
- Экстренные контакты стали редактируемыми и доступны прямо из `SOS`.

## Быстрый старт

### Требования

- Android Studio
- JDK 17
- Android SDK Platform 34
- Android SDK Build-Tools 34.0.0
- Android SDK Platform-Tools

### Клонирование

```bash
git clone https://github.com/gen1usup/pap_app.git
cd pap_app
```

### local.properties

Создайте `local.properties` в корне проекта и укажите путь к Android SDK.

Windows PowerShell:

```powershell
"sdk.dir=C\:\\Users\\$env:USERNAME\\AppData\\Local\\Android\\Sdk" | Out-File -FilePath local.properties -Encoding ascii
```

macOS / Linux:

```bash
echo "sdk.dir=$HOME/Library/Android/sdk" > local.properties
```

### Сборка

Windows:

```powershell
.\gradlew.bat :app:assembleDebug
```

macOS / Linux:

```bash
./gradlew :app:assembleDebug
```

APK будет создан в `app/build/outputs/apk/debug/app-debug.apk`.

## Этапы приложения

Поддерживаются три этапа:

- `PREPARING` / `До родов`
- `LABOR` / `Роды`
- `AFTER_BIRTH` / `После родов`

Принципы:

- этап меняется только вручную пользователем;
- ПДР не переводит приложение между этапами автоматически;
- этап влияет на главный экран, приоритет блоков, чек-листы и список быстрых действий.

Сменить этап можно:

- из drawer;
- из `Настроек`;
- через сценарные действия `Начались роды` и `Ребенок родился`.

## Логика ПДР

ПДР хранится в `Settings` и задается пользователем вручную.

ПДР используется только как мягкий контекст:

- на главной показывается сама дата;
- вычисляется количество дней до ПДР;
- ближе к ПДР поднимаются более релевантные карточки, например быстрый доступ к счетчику схваток.

ПДР не является автоматическим триггером смены этапа.

## Главный экран

`Главная` теперь отвечает на вопрос: что важно именно сейчас.

На экране показываются:

- текущий этап;
- ПДР и контекст по сроку;
- главная контекстная карточка с primary CTA;
- релевантный блок поддержки этапа;
- прогресс чек-листов текущего этапа;
- быстрые действия;
- последние события из журнала.

Поведение зависит от этапа:

- до родов наверху подготовка, чек-листы и мягкий доступ к сценариям;
- во время родов наверх поднимаются `События`, счетчик схваток, таймер вод и `SOS`;
- после родов главная перестраивается под документы, помощь маме и трекеры.

## События

Раздел `События` объединяет быстрые сценарии, которые нужны в реальном моменте:

- счетчик схваток;
- таймер вод;
- решение о выезде;
- `Начались роды`;
- `Выехали в роддом`;
- `Приехали в роддом`;
- `Ребенок родился`.

Сценарий `Ребенок родился`:

- переводит приложение в `AFTER_BIRTH`;
- фиксирует время рождения;
- позволяет сразу сохранить имя, вес и рост;
- допускает пропуск необязательных полей;
- создает запись в журнале.

## Журнал

`Журнал` хранит хронологию важных событий:

- схватки;
- воды;
- события родов;
- рождение ребенка;
- записи по первым дням после родов;
- ручные заметки.

Раздел остается отдельным от `Событий`: `События` нужны для быстрых действий, `Журнал` для истории.

## Чек-листы

Чек-листы разделены по этапам и сценариям.

### До родов

- документы;
- сумка в роддом для мамы;
- сумка для ребенка;
- сумка для отца;
- подготовка дома;
- машина и логистика;
- что делать, если роды начнутся.

### Когда начались роды

- что проверить сразу;
- что подготовить;
- что не забыть.

### Перед выездом

- документы;
- сумки;
- телефоны и зарядка;
- вода и ключи;
- безопасность дома;
- связь с роддомом.

### После родов

- документы ребенка;
- медицина и визиты;
- организация дома.

### Помощь маме

- еда и вода;
- отдых;
- быт;
- гости;
- помощь с ребенком.

Пользовательские пункты по-прежнему можно добавлять вручную.

## Экстренные контакты и SOS

Экстренные контакты хранятся локально в базе и редактируются на отдельном экране.

Поддерживаются слоты:

- скорая;
- роддом;
- врач;
- акушерка;
- доверенное лицо;
- жена / близкий человек;
- такси.

`SOS` использует эти контакты как источник быстрых звонков и при этом не перегружен медицинским текстом.

## Хранение данных

Приложение использует локальное хранение:

- `Room` для событий, чек-листов, трекеров и экстренных контактов;
- `DataStore` для настроек, темы, ПДР и текущего этапа приложения.

Текущая логика не требует сети для основной работы.

## Архитектурные принципы

- UI строится от состояния `ViewModel`.
- Бизнес-логика вынесена в `use case`.
- Данные этапа приложения централизованы в настройках.
- Навигация эволюционно доработана поверх текущей кодовой базы, а не переписана с нуля.
- Нерелевантные функции не удаляются, а становятся контекстными.
