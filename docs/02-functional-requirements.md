# Функциональные требования

## 1. Этапы приложения
Приложение поддерживает только три этапа:
- `PREPARING`
- `LABOR`
- `BABY_BORN`

Требования:
- этап хранится централизованно в настройках пользователя
- этап можно выбрать вручную через preview screen этапа в боковом меню
- этап можно выбрать вручную в настройках
- автоматические переходы поддерживаются только для ключевых milestone-событий

## 2. Переходы этапов
Обязательные правила:
- первая начатая схватка переводит приложение в `LABOR`
- запись вод переводит приложение в `LABOR`
- запись рождения переводит приложение в `BABY_BORN`
- при фиксации рождения активная схватка должна завершаться автоматически
- при фиксации рождения активный таймер вод должен закрываться автоматически
- история схваток и сессия не должны удаляться после рождения
- пользователь может вручную вернуться в `LABOR` после рождения

## 3. Главная
### `PREPARING`
`Главная` должна показывать:
- напоминание о ПДР при ее отсутствии
- подготовительные чек-листы
- информационные блоки
- последние события

### `LABOR`
`Главная` должна показывать:
- live-сводку `Активных родов`
- рекомендацию
- статус схватки
- статус вод
- быстрый доступ к контактам
- релевантный чек-лист

### `BABY_BORN`
`Главная` должна показывать:
- карточку ребенка всегда сверху
- приоритетные чек-листы после рождения
- остальные чек-листы
- последние события

## 4. Активные роды
Экран `Активные роды` должен:
- показывать короткую рекомендацию
- показывать текущий статус схватки
- показывать длительность и интервал
- показывать статус вод
- позволять начать или завершить схватку
- позволять зафиксировать воды
- позволять зафиксировать рождение

## 5. Рекомендации
Логика приоритета:
- если рождение зафиксировано, родовые рекомендации не являются первичными
- если воды не прозрачные, рекомендация срочная
- если воды прозрачные, рекомендация требует контакта с врачом или роддомом
- если воды не отходили, используется расчет по схваткам

## 6. События
Экран `События` должен:
- отражать текущий этап приложения
- давать доступ к действиям текущего этапа
- иметь отдельную кнопку заметки для текущего этапа
- сохранять заметки в журнал со stage-aware семантикой

## 7. Журнал
Запись журнала должна содержать:
- `timestamp`
- `stageAtCreation`
- `entryType`
- `type`
- `title`
- `description`

Фильтры журнала:
- `Все`
- `Подготовка`
- `Активные роды`
- `Ребенок родился`
- `Мои заметки`

## 8. Чек-листы
Экран `Чек-листы` должен:
- по умолчанию фильтроваться по текущему этапу
- позволять переключать фильтр этапа вручную
- показывать системные и пользовательские чек-листы
- позволять создавать пользовательский чек-лист
- позволять удалять любой чек-лист
- не восстанавливать удаленный системный чек-лист автоматически

## 9. Контакты
Экран `Контакты` должен:
- быть top-level разделом в нижней навигации
- содержать как минимум `Скорая помощь` и `Роддом`
- поддерживать редактирование телефонов
- хранить адрес роддома
- поддерживать звонок и построение маршрута

## 10. Удаления
Система должна поддерживать подтвержденное удаление:
- записи о водах
- отдельной схватки
- чек-листа
- пункта чек-листа

После удаления должна пересчитываться производная логика.
