# Правила отображения информации и состояний UI

Документ описывает, какие блоки показываются на экранах и при каких условиях.

## 1. Shell и навигация

### Bottom navigation

Bottom navigation показывается только на top-level маршрутах:

- `dashboard`
- `events`
- `checklist`
- `journal`

На вторичных экранах bottom bar скрыт.

### Drawer

Drawer доступен в shell-экранах и содержит:

- переключатель этапа;
- сервисные и вторичные разделы.

## 2. Главная

### Карточка текущего этапа

Показывается всегда.

Содержимое зависит от `appStage`:

- `PREPARING` — спокойная подготовка;
- `LABOR` — фокус на боевых сценариях;
- `AFTER_BIRTH` — послеродовой режим.

### Карточка ПДР

Показывается только если:

- `dueDate != null`.

### Главная контекстная карточка

Показывается всегда.

Кнопки зависят от этапа:

- `PREPARING`
  - `Начались роды`
  - `Чек-листы`
  - `Открыть счетчик схваток`, если активен `showContractionShortcut`
- `LABOR`
  - `События`
  - `Счетчик схваток`, если активен `showContractionShortcut`
  - `SOS`
- `AFTER_BIRTH`
  - `Трекеры`
  - `Помощь маме`
  - `Чек-листы`

### Условие `showContractionShortcut`

`PREPARING`:

- `true`, если есть активная сессия схваток;
- `true`, если ПДР задана и до нее `<= 21` дней;
- иначе `false`.

`LABOR`:

- всегда `true`.

`AFTER_BIRTH`:

- всегда `false`.

### Карточка “Добавьте данные о ребенке”

Показывается только если:

- текущий этап `AFTER_BIRTH`;
- и хотя бы одно из условий истинно:
  - `babyName` пусто;
  - `birthWeightGrams == null`;
  - `birthHeightCm == null`.

### Карточка чек-листов

Показывается всегда.

Данные берутся только для текущего этапа:

- `completedCount` — сумма отмеченных пунктов по чек-листам этапа;
- `totalCount` — сумма всех пунктов чек-листов этапа.

### Поддерживающая карточка этапа

Если `appStage == LABOR` и `showWaterBreakShortcut == true`:

- показывается карточка `Таймер вод`.

Если `appStage == AFTER_BIRTH`:

- показывается карточка `После родов`.

Во всех остальных случаях:

- показывается карточка `Когда ехать в роддом`.

### Условие `showWaterBreakShortcut`

- `true`, если `appStage == LABOR`;
- `true`, если есть активное событие вод;
- иначе `false`.

### Быстрые действия

Показываются всегда, но набор зависит от этапа.

`PREPARING`:

- Чек-листы
- Когда ехать в роддом
- Журнал
- Настройки

`LABOR`:

- События
- Таймер вод
- Когда ехать в роддом
- Журнал

`AFTER_BIRTH`:

- Трекеры
- После родов
- Помощь маме
- Журнал

### Последние события

Показываются всегда.

Источник:

- первые 4 события из общего журнала.

Если событий нет:

- показывается empty state.

## 3. Экран “События”

### Status card

Показывается всегда.

Дополнительные действия внутри карточки:

- кнопка открытия счетчика схваток — если есть активная сессия;
- кнопка таймера вод — если есть активное событие вод.

### Карточка быстрых инструментов

Показывается всегда.

Содержимое:

- счетчик схваток — только если текущий этап не `AFTER_BIRTH`;
- таймер вод — всегда;
- решение о выезде — всегда.

### Карточка “Начались роды”

Показывается только если:

- `appStage == PREPARING`.

### Карточка логистики родов

Показывается только если:

- `appStage == LABOR`.

Содержит:

- `Выехали в роддом`
- `Приехали в роддом`
- `Ребенок родился`

### Карточка сводки рождения

Показывается если:

- `appStage == AFTER_BIRTH`;
- или `laborSummary.birthTime != null`.

## 4. Экран “Чек-листы”

### Эффективный этап

Используется такой приоритет:

- если пользователь выбрал этап вручную внутри экрана, используется он;
- иначе используется `Settings.appStage`.

### Список чек-листов

Отображаются только чек-листы, где:

- `checklist.stage == effectiveStage`.

### Группировка

Чек-листы группируются по:

- `checklist.category`.

### Empty state

Показывается если:

- после фильтрации по этапу список пуст.

## 5. Экран “Журнал”

### Фильтр `ALL`

Показывает все события.

### Фильтр `LABOR`

Показывает только:

- `LABOR`
- `BIRTH`
- `CONTRACTION`
- `WATER_BREAK`

### Фильтр `POSTPARTUM`

Показывает только:

- `FEEDING`
- `DIAPER`
- `SLEEP`
- `NOTE`

### Форма добавления события

Для типа `LABOR` заголовок обязателен.

## 6. Экран “Таймер вод”

### Статусная карточка

Если `activeEvent != null`:

- заголовок сообщает об активном отсчете;
- показывается длительность;
- показываются время события и цвет;
- показывается кнопка закрытия активного события.

Если `activeEvent == null`:

- заголовок сообщает, что событие еще не зафиксировано;
- показывается предупреждающий краткий текст.

### Форма сохранения

Показывается всегда.

Содержит:

- выбор цвета через chip;
- поле заметок;
- кнопку сохранения.

### История

Если список истории пуст:

- empty state.

Иначе:

- карточки истории по каждому событию.

## 7. Экран “Счетчик схваток”

### Статусная карточка

Показывается всегда.

Основана на `recommendationLevel`:

- `MONITOR`
- `PREPARE`
- `GO_TO_HOSPITAL`
- `EMERGENCY`

### Большая круглая кнопка

Показывается всегда.

Состояния:

- нет сессии — старт первой схватки;
- сессия активна, схватка не идет — ожидание следующей;
- схватка идет — завершение текущей схватки.

### Кнопка завершения сессии

Показывается только если:

- есть активная сессия.

### График

Показывается всегда как карточка.

Если завершенных схваток нет:

- empty state внутри карточки.

Иначе:

- мини-график по последним завершенным схваткам.

### История

Показывает только завершенные схватки.

Порядок:

- от новых к старым.

## 8. Экран “Трекеры”

### Верхние вкладки

Показываются всегда:

- Кормления
- Подгузники
- Сон
- Заметки

### Содержимое формы и списка

Показывается только для выбранной вкладки.

Если по вкладке нет записей:

- empty state.

Иначе:

- список карточек записей.

## 9. Экран `SOS`

### Кнопка `Позвонить 112`

Показывается всегда.

### Кнопка `Позвонить в скорую / роддом`

Показывается всегда.

Номер берется:

- из контакта `MATERNITY_HOSPITAL`, если он заполнен;
- иначе используется `103`.

### Быстрые контакты

Показываются только для контактов, где:

- `phone.isNotBlank()`.

Если таких контактов нет:

- показывается только переход к управлению контактами.

## 10. Экран “Справка”

### Карточка роддома и маршрута

Показывается всегда.

Кнопка `Построить маршрут` показывается только если:

- адрес роддома не пустой.

Кнопка `Очистить адрес` показывается только если:

- адрес роддома не пустой.

## 11. Экран “Настройки”

### Диалог сброса

Показывается только если:

- `showResetDialog == true`.

### Карточки экрана

Показываются всегда:

- профиль и личные данные;
- этап приложения;
- тема;
- кнопка сохранения;
- кнопка полного сброса данных.

## 12. Экран “В родах”

### Карточка summary

Показывается всегда.

Содержит:

- текущее сохраненное время начала родов;
- кнопку установки времени начала родов на текущее значение;
- текущее сохраненное время рождения;
- кнопку установки времени рождения на текущее значение;
- поля имени, веса и роста;
- кнопку сохранения агрегированных данных.

### Обратная связь

После успешного сохранения пользователь получает snackbar с подтверждением.

### Карточка ручного события родов

Показывается всегда.

Содержит:

- поле заголовка события;
- поле заметки;
- кнопку добавления события в журнал.

### Список событий родов

Показывает только события журнала типов:

- `LABOR`
- `BIRTH`

Если таких событий нет:

- empty state.

## 13. Экран “После родов”

Экран показывает статические информационные секции по послеродовому периоду.

Он не зависит от локальных логов и не содержит собственной формы ввода.

## 14. Экран “Помощь маме”

Экран показывает статические подсказки по категориям помощи.

Содержимое берется из ресурсов и не зависит от локального состояния данных.

## 15. Экран “О приложении”

Экран показывает статические информационные карточки:

- offline-first подход;
- локальное хранение данных;
- отказ от медицинских обещаний.
