# Правила отображения информации и UI-состояний

## 1. Shell и навигация

### Bottom navigation

Bottom navigation показывается на top-level маршрутах:

- `dashboard`
- `events`
- `checklist`

Состав:

- `Главная`
- `События`
- `Чек-листы`

### Журнал

`Журнал` не является bottom-tab.

Он открывается через top-bar action на экранах:

- `Главная`
- `События`
- `Чек-листы`

### Drawer

Drawer разделен на группы:

- `Этапы`
- `Сервис`

После рождения пункты `Подготовка` и `Схватки` визуально блокируются и не активируются.

## 2. Главная

## Reminder о ПДР

Показывается, если `dueDate == null`.

Содержит:

- объяснение, зачем нужна ПДР
- CTA для перехода в настройки

## До 37 недель

При `PREPARING` и сроке раньше 37 недель показываются только подготовительные блоки.

## С 37 недель

При `PREPARING` и readiness-window сохраняется подготовительный контент и дополнительно показываются:

- hospital readiness
- вход в счетчик схваток

Shortcut к счетчику не уходит наверх и не вытесняет подготовку.

## Во время схваток

Если этап `CONTRACTIONS` или есть активная сессия схваток, порядок на `Главной` такой:

1. live-блок схваток
2. блок таймера вод
3. спокойный блок быстрых действий
4. релевантные чек-листы

### Live-блок схваток

Показывает:

- идет ли сейчас схватка
- текущий интервал
- рекомендацию
- действие `Начать схватку` или `Завершить схватку`

Для рекомендаций используется тот же источник логики, что и на отдельном экране схваток.

### Быстрые действия в родах

Показывают:

- открытие контактов
- маршрут до роддома
- звонок врачу
- звонок в роддом

## После рождения

После `birthTime != null`:

- live-блок схваток скрывается
- этап уходит в `AT_HOSPITAL`
- при незаполненных birth details показывается hospital-карточка данных родов

## После `Приехали домой`

После перехода в `AT_HOME`:

- родовые боевые блоки исчезают
- поднимаются домашние действия, трекеры и чек-листы дома

## 3. События

Экран `События` stage-aware.

### `PREPARING`

Показывает:

- подготовительные заметки
- готовность сумки
- test drive
- practice/start contraction timer

### `CONTRACTIONS`

Показывает:

- схватки
- воды
- выезд / приезд
- начало родов
- labor notes

### `AT_HOSPITAL`

Показывает:

- рождение
- support actions
- hospital notes
- переход домой

### `AT_HOME`

Показывает:

- кормление
- сон
- подгузник
- температура
- вес
- заметки

## Активный таймер вод

Если таймер вод остается активным в `AT_HOSPITAL` или `AT_HOME`, пользователь все равно видит явный путь к инструменту через `Events` и соответствующий shortcut.

## 4. Чек-листы

По умолчанию экран фильтруется по текущему этапу приложения.

Пользователь может вручную выбрать любой другой этап через chips.

Поддерживаются:

- системные списки
- пользовательские списки

Для пользовательских списков доступны:

- создание
- переименование
- удаление

Для пунктов доступны:

- добавление
- удаление
- отметка done / undone

## 5. Журнал

### Общая история

Элементы журнала по умолчанию компактные.

Если у события есть подробности, карточка раскрывается по нажатию.

Для contraction events в раскрытии показываются:

- старт
- конец
- длительность
- интервал

### Семантические фильтры

Labor/hospital notes не теряются из-за generic `NOTE`:

- labor notes видны в labor-контексте
- hospital notes видны в hospital-контексте
- home notes видны в postpartum/home-контексте

## 6. Контакты и SOS

### Контакты

По умолчанию существует только `112`.

Остальные контакты добавляются пользователем вручную.

Контакт роддома может хранить:

- телефон
- адрес

И поддерживает:

- звонок
- маршрут
- копирование адреса

### SOS

Всегда показывает:

- `Позвонить 112`
- тревожные признаки
- быстрый переход к контактам

Если добавлены врач или жена, появляются отдельные быстрые кнопки звонка.
