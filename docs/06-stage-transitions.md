# Правила переходов между этапами

## Актуальные этапы
- `PREPARING`
- `LABOR`
- `BABY_BORN`

## Автоматические переходы
### Схватка
Условие:
- пользователь запускает первую схватку

Результат:
- этап становится `LABOR`

### Воды
Условие:
- пользователь сохраняет запись о водах

Результат:
- этап становится `LABOR`

### Рождение
Условие:
- пользователь фиксирует рождение

Результат:
- этап становится `BABY_BORN`
- активная схватка завершается
- активный таймер вод закрывается
- история схваток и событий сохраняется

## Ручные переходы
Ручной переход делается:
1. через боковое меню
2. затем через preview screen выбранного этапа
3. затем через подтверждающую кнопку

Ручной возврат в более ранний этап допускается.
Это нужно для реальных сценариев тестирования и повторного просмотра родового контекста.

## Что не переводит этап автоматически
- ПДР
- срок беременности сам по себе
- расчеты аналитики схваток
- открытие экрана
- просмотр журнала

## Совместимость старых значений
Старые сохраненные значения читаются так:
- `CONTRACTIONS` -> `LABOR`
- `LABOR` -> `LABOR`
- `AT_HOSPITAL` -> `BABY_BORN`
- `AT_HOME` -> `BABY_BORN`
- `AFTER_BIRTH` -> `BABY_BORN`
