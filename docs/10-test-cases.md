# Тест-кейсы проекта

Документ описывает все автотесты, которые есть в проекте на текущий момент.

Для каждого теста зафиксированы:

- уровень тестирования;
- кодовый источник;
- позитивный или негативный сценарий;
- входные условия;
- ожидаемый результат.

## 1. Правила прогона

Локально запускаются:

- JVM unit tests из `app/src/test/java`

На эмуляторе или телефоне запускаются:

- instrumented/UI tests из `app/src/androidTest/java`

Назначение:

- unit tests проверяют бизнес-логику, границы и защиту от некорректных данных;
- instrumented tests проверяют реальные пользовательские сценарии, отображение экранов и ошибки через настоящий UI.

### 1.2 Как тестируются долгие сценарии схваток

Для проверок, где в продукте логика зависит от 20-90 минут истории схваток, тесты не ждут реальное время.

Вместо этого используется тестовый сидер [TestAppStateSeeder.kt](c:/PROJECTS/pap_app/app/src/androidTest/java/com/dadnavigator/app/testsupport/TestAppStateSeeder.kt), который перед запуском приложения:

- очищает локальные данные;
- сохраняет нужный этап приложения;
- записывает в `DataStore` и Room детерминированное состояние;
- создает активную сессию схваток;
- добавляет завершенные схватки с заранее заданными `startedAt/endedAt`.

За счет этого instrumented-тест может за секунды проверить UI для сценариев:

- нерегулярные схватки, которые должны остаться в режиме наблюдения;
- регулярные умеренные схватки, которые должны дать рекомендацию готовиться к выезду;
- регулярные сильные схватки, которые должны дать рекомендацию ехать в роддом.

## 4. Что покрыто этими тестами

Покрываются:

- навигация top-level и переходы из drawer;
- корректная смена этапов;
- позитивные сценарии открытия основных экранов;
- inline-валидация поля ПДР на Compose-экране;
- быстрые UI-проверки долгих сценариев схваток без ожидания реального времени;
- негативные сценарии валидации пользовательского ввода;
- защита от дублей в событиях начала родов и рождения;
- граничные и критические сценарии в рекомендациях по схваткам и решении о выезде;
- сервисные экраны `SOS`, `Справка`, `Настройки`, `Журнал`.

## 5. Что пока не покрыто автоматически

Пока отдельно не автоматизированы:

- полный e2e сценарий `схватки -> воды -> рождение -> трекеры` через UI;
- ввод и редактирование экстренных контактов;
- послеродовые трекеры на всех вкладках;
- экспорт/внешние интеграции;
- сравнение светлой и темной темы на уровне screenshot regression.

Эти сценарии можно добавить следующей итерацией либо как новые instrumented tests, либо как snapshot/UI regression suite.
