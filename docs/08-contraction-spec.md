# Спецификация счетчика схваток

## Назначение

Счетчик схваток нужен для быстрого ручного учета схваток без перегруженного интерфейса.

Он должен позволять:

- начать и завершить схватку одним крупным элементом;
- видеть текущую рекомендацию;
- понимать длительность и интервалы;
- сохранить историю сессии.

## Модель данных

Сценарий строится на двух сущностях:

- `ContractionSession`
- `Contraction`

Одна сессия содержит несколько схваток.

## Основные состояния UI

### 1. Сессии еще нет

Показывается круглый CTA с действием запуска первой схватки.

### 2. Сессия активна, но схватка не идет

Показывается ожидание следующей схватки и кнопка начала новой записи.

### 3. Идет активная схватка

Второе нажатие по кругу завершает ее.

## Отображаемые показатели

Экран показывает:

- длительность текущей сессии;
- количество схваток;
- текущую или последнюю длительность;
- последний интервал;
- средние значения;
- историю завершенных схваток;
- мини-график.

## Рекомендации

В статусной карточке используется один из уровней:

- `MONITOR`
- `PREPARE`
- `GO_TO_HOSPITAL`
- `EMERGENCY`

Отображение зависит от вычислений use case аналитики, а не от ручного выбора пользователя.

## Правила на главной

Shortcut в счетчик схваток:

- не показывается автоматически до 37 недели, если сессия еще не активировалась;
- появляется в `PREPARING` с 37 недели или при активной сессии;
- всегда доступен в `CONTRACTIONS`;
- скрывается в `AT_HOSPITAL` и `AT_HOME`.

## Что счетчик не делает

Счетчик не:

- меняет этап автоматически;
- не переводит приложение сам по себе в `CONTRACTIONS`;
- не переводит приложение в `AT_HOSPITAL` или `AT_HOME`;
- не заменяет медицинское решение.

## Тестирование долгих сценариев

Долгие сценарии проверяются без реального ожидания времени.

Для этого instrumented tests используют [TestAppStateSeeder.kt](c:/PROJECTS/pap_app/app/src/androidTest/java/com/dadnavigator/app/testsupport/TestAppStateSeeder.kt), который напрямую записывает в локальную базу:

- активную сессию;
- завершенные схватки с нужными интервалами и длительностями;
- нужный этап приложения.

Это позволяет быстро проверить переходы рекомендаций:

- `Наблюдаем`
- `Готовимся к выезду`
- `Пора ехать`
